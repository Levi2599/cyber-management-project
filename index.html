🔗 חיבור אוטומטי ל-Repository (סביבת ייצור)

חיבור המאגר (Repository) ל-n8n מאפשר למערכת לזהות ולתקן חולשות אבטחה אוטומטית ברקע. במקום להעתיק ולהדביק קטעי קוד, המערכת מאזינה לאירועי קוד ב-GitHub ומבצעת את התיקון ישירות בתוך ה-Pull Request.

  • 1 העתקת כתובת ה-Webhook:
    זוהי כתובת היעד שבה שרת ה-n8n מאזין לאירועים מ-GitHub. העתק את הכתובת הבאה:
    http://35.193.190.149:5678/webhook/github-webhook
    * הערה: במידה ואתם עובדים בסביבה מקומית והמאגר ב-GitHub אינו מותקן על אותו מחשב, יש להשתמש בכתובת מנהרה ציבורית (כגון ngrok או localtunnel) כדי ש-GitHub יוכל לגשת ל-localhost שלכם.
  • 2 הוספת ה-Webhook במאגר ה-GitHub שלכם:
    בצעו את השלבים הבאים בממשק של GitHub:
    1. היכנסו לעמוד המאגר שלכם ב-GitHub ולחצו על לשונית Settings (הגדרות) בתפריט העליון.
    2. בתפריט הצידי השמאלי, לחצו על Webhooks.
    3. לחצו על כפתור Add webhook בצד ימין למעלה.
    4. בשדה Payload URL, הדביקו את כתובת ה-Webhook שהעתקתם בשלב 1.
    5. בשדה Content type, בחרו באפשרות application/json.
    6. תחת "Which events...", בחרו באפשרות Let me select individual events, וסמנו בוי (V) את Pushes ו- Pull Requests.
    7. לחצו על הכפתור הירוק Add webhook בתחתית הטופס כדי לשמור.
  • 3 סריקה אקטיבית ותיקון אוטומטי (Self-Healing):
    זהו זה! החיבור הושלם. מעתה, בכל פעם שיווצר Pull Request חדש או שיתבצע Push במאגר:
    • סוכן ה-Red Team ישלוף אוטומטית את שינויי הקוד (Diff) ויכתוב דוח פגיעות כהערה (Comment) בתוך ה-PR.
    • סוכן ה-Blue Team ייצר קוד מתוקן ויבצע Commit מתקן אוטומטי ישירות לענף הרלוונטי.

⚡ סימולטור פשוט לבדיקת החיבור מול GitHub

כדי לוודא ששרת ה-n8n והסוכנים שלכם מוגדרים נכון מול ה-API של GitHub, הדביקו קישור ל-Pull Request פתוח במאגר שלכם (הכולל קטע קוד פגיע, למשל SQL Injection) ולחצו על הכפתור. הסימולטור ישלח אירוע Webhook מדומה ל-n8n שיפעיל את תהליך הסריקה והתיקון ב-GitHub.
